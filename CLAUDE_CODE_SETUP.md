# 🚀 Настройка Claude Code для Meal Planner

Полная пошаговая инструкция по интеграции проекта с Claude Code в VSCode.

## ✅ Требования

- Node.js 18+
- VSCode 1.98.0 или выше
- Claude Pro, Max, Team, Enterprise или API ключ
- Go 1.21+
- Docker & Docker Compose

## 📋 Шаг 1: Подготовка VSCode

### 1.1. Убедись что всё установлено

```bash
# Проверка Node.js
node --version  # Должен быть v18+

# Проверка Go
go version     # Должен быть 1.21+

# Проверка VSCode версии
code --version # Должна быть 1.98.0+
```

Если что-то не установлено, посети официальные сайты для скачивания.

### 1.2. Скачай все файлы проекта

Создай папку и положи туда все файлы из `meal-planner/`:

```bash
mkdir meal-planner
cd meal-planner

# Скопируй сюда все файлы:
# - cmd/server/main.go
# - internal/...
# - migrations/...
# - frontend/...
# - docker-compose.yml
# - go.mod
# - .env.example
# - CLAUDE.md
# - .claudeignore
```

### 1.3. Создай .env файл

```bash
cp .env.example .env
```

Содержимое должно быть правильное (тестовые ключи уже есть).

## 📍 Шаг 2: Установка Claude Code Extension

### 2.1. Установи расширение в VSCode

Нажми Ctrl+Shift+X (Windows/Linux) или Cmd+Shift+X (Mac) чтобы открыть Extensions

Затем:
1. Поиск: `Claude Code`
2. Найди расширение от **Anthropic** (2M+ installs, синяя галочка)
3. Нажми **Install**

### 2.2. Первый запуск

После установки:
1. Ты увидишь Spark ⚡ иконку в левом сайдбаре
2. Нажми на неё → откроется Claude Code панель
3. Первый раз попросит авторизацию через браузер
4. Выбери: "Claude Pro" или "API Key" в зависимости от твоего плана

### 2.3. Выбери режим работы

Claude Code предлагает два режима:

- **Plan Mode** ✅ (рекомендуется для начала)
  - Показывает план перед изменениями
  - Ты подтверждаешь каждое действие
  - Полный контроль над процессом

- **Auto-Accept Mode**
  - Claude автоматически применяет изменения
  - Быстрее, но менее безопасно
  - Используй когда уверен в своём коде

### 2.4. Сообщи Claude Code о проекте

Когда Claude Code откроется:

```
Привет! Я работаю над проектом Meal Planner.
Это сервис подбора блюд по калориям, написан на Go.
Вот контекст: [скопируй содержимое CLAUDE.md]
```

Или просто скажи:

```
Прочитай CLAUDE.md для контекста проекта
```

Claude Code автоматически прочитает файл и поймет структуру.

## 🏃 Шаг 3: Запуск проекта локально

Важно: проект должен работать перед тем как использовать Claude Code!

### 3.1. Запусти PostgreSQL

В отдельном терминале:

```bash
cd meal-planner
docker-compose up -d

# Проверь что работает
docker-compose ps
```

### 3.2. Установи зависимости Go

```bash
go mod download
go mod tidy
```

### 3.3. Запусти сервер

```bash
cd cmd/server
go run main.go

# Должен вывести:
# ✓ Connected to database
# ✓ Executed migration: 001_create_tables.sql
# ✓ Executed migration: 002_seed_test_data.sql
# 🚀 Starting Meal Planner API
# ✓ Server listening on http://localhost:8080
```

### 3.4. Проверь что фронтенд работает

Открой в браузере:
```
http://localhost:8080
```

Должен загрузиться интерфейс входа.

## 💬 Шаг 4: Использование Claude Code

### 4.1. Как задавать задачи

Используй Command Palette (Cmd+Shift+P или Ctrl+Shift+P) чтобы открыть Claude Code команды

Примеры команд:

```
# Добавить новый endpoint
Добавь GET /api/meals/:id который возвращает одно блюдо по ID

# Исправить баг
Ошибка 401 при попытке подтвердить API ключ. Помогите найти проблему

# Улучшить код
Рефактори internal/api/handlers.go чтобы код был более читаемым

# Добавить фичу
Добавь фильтр по цене к эндпоинту /api/suggest
```

### 4.2. Рабочий процесс

1. **Напиши задачу** в Claude Code панели
2. **Просмотри plan** — Claude покажет что он собирается делать
3. **Одобри изменения** — нажми "Accept" или пересмотри код
4. **Проверь результат** — вернись к коду и посмотри diffs
5. **Протестируй** — убедись что всё работает

### 4.3. Примеры задач для Claude Code

#### Пример 1: Добавить новый API endpoint

```
Добавь в API новый endpoint:

Путь: GET /api/user-stats
Описание: Возвращает статистику пользователя (кол-во сохраненных наборов, всего блюд съедено)

Response:
{
  "total_collections": 10,
  "total_meals_saved": 25,
  "average_calories": 1250
}

Требования:
- Защищен аутентификацией (требует X-API-Key)
- Query из meal_collections
- Вернуть JSON
```

#### Пример 2: Исправить баг

```
При попытке сохранить набор получаю ошибку:
"pq: invalid input syntax for type uuid"

Вот стек трейс: [скопируй ошибку]

Помоги найти проблему и исправить её
```

#### Пример 3: Улучшить фронтенд

```
Добавь на фронтенд фичу "Быстрый поиск" (Quick Search):
- Input поле вверху страницы
- Поиск по названию блюда в реальном времени
- Показывает только блюда из выбранного ресторана
- Кликнул на блюдо → добавляется в текущий набор
```

### 4.4. Советы для лучшего взаимодействия

✅ **Делай:**
- Будь конкретен в описании (не "добавь фичу", а "добавь сортировку по цене")
- Скопируй ошибки полностью (не только сообщение)
- Попроси Claude Code объяснить код если не понимаешь
- Тестируй изменения сразу после применения

❌ **Не делай:**
- Не просирай слишком большие задачи (раздели на части)
- Не игнорируй errors/warnings — спроси что не так
- Не коммитай код в git если не протестировал

## 🔄 Частые команды Claude Code

| Действие | Команда |
|----------|---------|
| Новый чат | Ctrl+N (Windows) или Cmd+N (Mac) |
| Переключить фокус | Ctrl+Esc или Cmd+Esc |
| Просмотреть команды | Ctrl+Shift+P → "Claude Code" |
| Отправить промпт | Enter |
| Новая строка в промпте | Shift+Enter |

## 🧪 Тестирование после изменений

После каждого изменения:

```bash
# 1. Убедись что код компилируется
cd cmd/server
go build -o meal-planner .

# 2. Перезапусти сервер (если не упал)
go run main.go

# 3. Тестируй API через curl
curl -H "X-API-Key: test-user-abc123xyz" \
  http://localhost:8080/api/restaurants
```

Если что-то сломалось → покажи ошибку Claude Code и он поможет исправить.

## 🐛 Решение проблем с Claude Code

| Проблема | Решение |
|----------|---------|
| **Claude Code не видит мой код** | Убедись что .claudeignore правильный, перезагрузи VSCode |
| **Изменения не применяются** | Проверь что находишься в `meal-planner/` папке |
| **Ошибка 401 при запросе API** | Убедись что сервер работает (`http://localhost:8080/health`) |
| **Claude Code очень медленный** | Вероятно большой контекст — уменьши .claudeignore |
| **Не могу авторизоваться** | Перезагрузи VSCode, попробуй снова |
