4. Перезагрузи VSCode

## 🎯 Примеры использования Claude Code

### Пример 1: Добавить простой endpoint (Task 1.1)

```
Добавь новый endpoint:

GET /api/restaurants/:id

Что должен делать:
- Возвращает ресторан + все блюда
- Требует аутентификацию X-API-Key

Тип ответа: JSON с ресторанольд и массивом блюд
```

Claude Code автоматически:
- Добавит метод в handlers.go
- Добавит query в queries.go (если нужно)
- Зарегистрирует маршрут
- Покажет дифф для проверки

### Пример 2: Исправить баг

```
При сохранении набора получаю ошибку:
database/sql: invalid syntax for UUID

Вот stack trace: [скопируй полную ошибку]

Помоги найти и исправить проблему
```

Claude Code:
- Прочитает код
- Найдет проблему
- Покажет исправление
- Объяснит что было не так

### Пример 3: Добавить фичу с деталями

```
Задача: Добавить фильтр по цене

Требования:
1. Обновить /api/suggest чтобы принимал max_price (опционально)
2. В алгоритме фильтровать результаты по цене
3. На фронтенде добавить input для max_price
4. Показывать общую цену каждого набора

Response должен содержать total_price для каждого набора
```

Claude Code сделает:
- Backend изменения в handlers, queries, algorithm
- Frontend изменения в HTML/JS
- Покажет всё в виде diffs для проверки

## 💡 Полезные советы

### ✅ Правильный способ работать

1. **Дай ясную задачу**
   ```
   "Добавь эндпоинт GET /api/meal/:id"
   ❌ Вместо: "Добавь что-нибудь новое"
   ```

2. **Скопируй ошибки полностью**
   ```
   [полный stack trace или error message]
   ❌ Вместо: "Что-то сломалось"
   ```

3. **Тестируй после каждого изменения**
   ```bash
   # Перезапусти сервер
   go run cmd/server/main.go
   
   # Тестируй через curl или UI
   ```

4. **Проси объяснения**
   ```
   "Объясни как работает knapsack алгоритм"
   "Почему мы используем backtracking?"
   ```

### ❌ Ошибки которых избегать

- ❌ "Добавь все фичи сразу" → раздели на части
- ❌ "Исправь всё" без описания проблемы → будь конкретен
- ❌ Не коммитить код если не протестировал
- ❌ Игнорировать предупреждения — спроси что не так
